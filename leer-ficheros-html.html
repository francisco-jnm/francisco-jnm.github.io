Los ficheros HTML (que son páginas web) se leen en R utilizando el paquete XML. Este paquete, como su nombre indica, está pensado para tratar ficheros XML pero también incluye funciones para HTML.

El lenguaje HTML es parecido pero mucho menos estructurado que el XML. Por eso, obtendremos unos datos menos limpios y habrá que trabajar mas para limpiarlos.

El fichero “baloncesto.html” contiene datos de los 50 mejores anotadores en la temporada regular 2016-17 de la euroliga de baloncesto.

Queremos construir un marco de datos extrayendo del fichero html la información de: jugador, equipo y puntos.

Leemos el fichero html con htmlParse y cargamos el nodo raíz con xmlRoot.

library("XML")
html <- htmlParse(file = "baloncesto.html")
htmltop <- xmlRoot(html)

La función básica es xpathSApply.

Por inspección directa del fichero vemos que los jugadores están en un nodo “span” con atributo “class” con valor “hidden-xs”:

Los puntos anotados no son tan fáciles de localizar, no hay un nodo con un valor concreto de uno de sus atributos que los identifique.
