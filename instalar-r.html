Git

Git es un sistema de control de versiones de proyectos. Generalmente, nuestro proyecto estará en un directorio raíz del que colgarán diversos subdirectorios y archivos. Con Git, podemos crear una copia oculta del directorio raiz y etiquetarla como “versión 0” o similar. De esta manera, tendremos un histórico de nuestro proyecto y siempre podremos volver a una versión concreta, crear “ramas” para avanzar el proyecto en diferentes direcciones, mezclar elementos de diversas versiones y ramas, etc…

GitHub

Web de GitHub: https://github.com/

GitHub es una nube donde almacenar proyectos, cada almacén se llama “repositorio”. Lo que le hace particular es que se pueden gestionar esos proyectos con Git, con lo que se tiene un histórico de versiones on-line y otros programadores pueden crear sus “ramas” de nuestro proyecto, mezclar elementos de diversas ramas y versiones, etc.

GitHub Desktop

Descarga de GitHub Desktop: https://desktop.github.com/
Se hace la instalación por defecto.

La gestión de GitHub directamente con Git es demasiado espartana. En la práctica se utiliza “GitHub Desktop” que interacciona visualmente con Git.

Con GitHub Desktop instalado, el uso básico de GitHub es como sigue:

  • Para empezar un proyecto: creamos un repositorio en GitHub y pulsamos “Clone or download > Open in desktop”. GitHub Desktop nos pedirá un directorio local y quedará asociado al repositorio.

  • Cada vez que tengamos el proyecto en un estado que valga la pena guardar: abrimos “GitHub desktop” y ponemos un nombre descriptivo en el campo “summary” del proyecto. Pulsamos “Commit to master” y luego “Sync”. Se subirá todo a la web de “GitHub” y quedará como una versión del proyecto.

  • Cuando nos interesa un repositorio de alguien: vamos a GitHub y entramos en el repositorio. Pulsamos en “Fork” y se crea una copia en nuestra cuenta. Pulsamos “Clone or download > Open in desktop”. GitHub Desktop nos pedirá un directorio local y quedará asociado al repositorio.
