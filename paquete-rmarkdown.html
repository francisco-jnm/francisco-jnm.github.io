Documentos “mágicos” con código que se autoejecuta.

La investigación reproducible sólo es factible si el investigador se molesta en hacer un documento en el que se mezclen las explicaciones verbales con el código. Además de hacer este documento, el investigador tiene que dejar accesibles los conjuntos de datos con los que trabaja el código.

Si el investigador hace eso, cualquiera podrá reproducir la investigación, cuestionarla, ampliarla, sugerir alternativas…

Existen diversas herramientas para producir estos documentos mixtos (texto y código que se autoejecuta) y la herramienta propia de R es el paquete RMarkDown.

Estas líneas que estamos leyendo se han escrito con RMarkDown.

Instalación de RMarkDown

RMarkDown se instala como cualquier otro paquete de RStudio:

install.packages("rmarkdown")

En cuanto se instala, ya tenemos en el menú “File > New File” un submenú “RMarkDown”.

Básicos de RMarkDown (texto)

Poniendo un asterisco al principio y al final de un texto, aparece en cursiva

Poniendo dos asteriscos al principio y al final de un texto, aparece en negrita.

Se pueden crear listas con los símbolos: "*" “+” “-”

* Málaga
* Almería
* Granada
  • Málaga
  • Almería
  • Granada

Se pueden crear sublistas utilizando el tabulador:

* Andalucía  
  * Málaga  
  * Almería  
  * Granada  
  * ...
  • Andalucía
    • Málaga
    • Almería
    • Granada
    • …

También se pueden hacer listas numeradas utilizando “1.” como símbolo. RMarkDown cambiará el “1.” por el número que corresponda. Se puede utilizar cualquier otro número o, directamente, indicar bien la numeración.

1. Málaga  
1. Almería  
1. Granada
  1. Málaga
  2. Almería
  3. Granada

Básicos de RMarkDown (código)

El código básico (autoejecutable) se escribe así:

{r}
library(dplyr)

Para evitar que se impriman mensajes y avisos:

{r message = FALSE, warning = FALSE}
library(dplyr)

Para no mostrar el código pero lanzar algún cálculo:

{r message = FALSE, warning = FALSE, echo = FALSE}
muestra <- rnorm(10, 4, 1.5)
media <- mean(muestra)

Esto es útil para mostrar los resultados más adelante en el texto:

r media

3.8128494

Archivos que puede producir RMarkDown

Cuando se pulsa el botón “Knit”, se genera un archivo del formato indicado al principio del documento con el comando output.

Formato markdown (extensión “.md”)

---
output: github_document
---

Formato html (extensión “.html”)

---
output: html_document
---

Formato pdf (extensión “.pdf”)

---
output: pdf_document
---

Formato Microsoft Word (extensión “.doc”)

---
output: word_document
---

El formato básico es “md” (markdown) y siempre se genera un archivo md que luego se convierte al formato solicitado. Tras la conversión, se elimina automáticamente el archivo “md”, a menos que hayamos indicado “keep_md: yes”.

---
output:
  html_document:
    keep_md: yes
---

Rpubs: una red social para compartir archivos RMarkDown

Los creadores de RStudio han creado también la web Rpubs donde se puede subir cualquier archivo creado con RMarkDown. El resto de usuarios puede comentar y compartir el documento.

La web está totalmente integrada en RStudio, se sube el documento pulsando el botón “Publish” después de generar el documento con el botón “Knit”.

BookDown: una mejora de RMarkDown (libros y artículos más profesionales)

El paquete bookdown se puede instalar a continuación de RMarkDown y permite escribir libros y artículos más profesionales.

Además, los creadores de BookDown han seguido el ejemplo de Rpubs y permiten mostrar gratuitamente los libros escritos con este paquete en la web de bookdown.
